# portalm-leyu-hub

这是一个用于归档和发布多个独立 HTML 页面的仓库。仓库内收录的页面彼此独立，不绑定任何特定域名或网站，主要作为静态 HTML 素材的集中存储与版本管理。

## 项目简介

随着前端页面开发与测试工作的开展，常需要维护多个临时或独立的 HTML 页面。本仓库旨在提供一个统一存放空间，便于归档不同阶段的页面快照、组件示例、原型演示等文件，同时利用 GitHub Pages 等静态托管能力实现简单发布。

仓库内容以纯 HTML、CSS、JavaScript 文件为主，不涉及后端逻辑或数据库依赖。

## 目录结构说明

```
portalm-leyu-hub/
├── pages/               # 存放独立的 HTML 页面文件
│   ├── example-01.html
│   ├── example-02.html
│   └── ...
├── assets/              # 页面引用的公共资源（如样式、脚本、图片等）
│   ├── css/
│   ├── js/
│   └── images/
├── docs/                # 项目文档或说明性文件（可选）
└── README.md            # 本文件
```

- `pages/`：主要存放独立 HTML 文件，每个文件可视为一个独立页面。
- `assets/`：页面可能共用的资源文件，按类型分目录存放。
- `docs/`：用于存放与项目相关的说明文档或笔记。

## 页面归档说明

- 每个 HTML 页面均为自包含或仅引用 `assets/` 下资源的独立文件。
- 页面命名建议采用有意义的英文或拼音，便于识别。
- 归档时请保持页面完整性，确保资源引用路径正确。
- 如需引用外部库（如 CDN），请在页面中明确标注来源。

## 维护说明

- 仓库主要面向个人或小团队维护，欢迎通过 Issue 或 Pull Request 提出建议。
- 新增页面请放入 `pages/` 目录，并在必要时更新本 README 的目录结构示例。
- 不建议在仓库中存放包含敏感信息或违反平台政策的页面。
- 本仓库不提供自动部署脚本，如需发布请自行配置 GitHub Pages 或其他静态托管服务。
